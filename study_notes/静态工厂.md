## 静态工厂

静态工厂就是一个个静态方法，可以帮助快速创建对象

```// ❌ 没有工厂方法——每次都要手动写一堆
  Message ping = new Message(
      MessageType.PING,
      UUID.randomUUID().toString(),   // 每次都要生成 UUID
      ""                               // PING 没有 body，但要传空字符串
  );
    Message task = new Message(
      MessageType.TASK_REQUEST,
      UUID.randomUUID().toString(),   // 又要写一遍 UUID
      "什么是虚拟线程？"
  );

  // ✅ 有工厂方法——一行搞定
  Message ping = Message.createPing();
  Message task = Message.createTaskRequest("什么是虚拟线程？");
```



  为什么叫**「工厂」**？ 就像工厂生产产品一样，这个方法负责「生产」Message 对象。它是 static 的，所以不用  先 new 一个 Message出来才能调用——直接用类名调用：Message.createPing()。



  这就是全部了——工厂方法 = 封装了 new 的细节，让创建对象更简单、更不容易出错。
