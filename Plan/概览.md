📅 一个月（4周）特训学习与开发任务卡
🧱 第1周：打牢地基，搞定高性能通信引擎
本周目标：让两台电脑（或者两个进程）能用最快的速度互相发送二进制数据，并用上Java 21的最新特性。

Day 1-2：拥抱 Java 21 与 虚拟线程 (Virtual Threads)

学习任务：了解什么是虚拟线程，它和传统线程池有什么区别。为什么它能解决高并发I/O的阻塞问题。

代码任务：写一个简单的Echo服务器，用传统线程池实现一次，再用 Executors.newVirtualThreadPerTaskExecutor() 改造一次，感受其轻量级。

Day 3-5：Netty 基础与自定义协议

学习任务：学习 Netty 的核心组件（EventLoop, Channel, Pipeline, ByteBuf）。理解什么是“零拷贝（Zero-Copy）”。

代码任务：不使用现成的HTTP，自己定义一个简单的二进制协议（比如：消息头包含长度、类型，消息体是内容）。基于Netty写一个长连接的 Client 和 Server，实现互相发送自定义消息。

Day 6-7：封装 RPC (远程过程调用) 骨架

代码任务：把前几天的Netty通信封装一下，做到“在A机器上调用一个方法，实际上是发消息让B机器执行并返回结果”。这就是未来中枢节点给计算节点发指令的基础。

🧠 第2周：打造最强大脑，Controller中枢节点与调度
本周目标：实现中枢管理节点，能够管理多个计算节点的加入，并能超高速地派发任务。

Day 1-2：集群节点发现与心跳机制

学习任务：了解心跳保活（Keep-Alive）的概念。

代码任务：让第1周写的 Client（模拟计算节点）定时向 Server（模拟Controller）发送“我还活着”的心跳包。Server 维护一个在线节点列表（ConcurrentHashMap），超时没收到心跳就踢掉节点。

Day 3-4：哈希环与任务调度策略

学习任务：学习“一致性哈希（Consistent Hashing）”算法。为什么不用简单的取模来分配任务？

代码任务：写一个一致性哈希工具类。当有新任务（比如图片识别任务）来时，根据任务的ID算出它应该被派发给哪个在线的 Compute Worker。

Day 5-7：Disruptor 无锁环形缓冲区引入

学习任务：并发编程进阶。学习 LMAX Disruptor 为什么比 Java 自带的 ArrayBlockingQueue 快（伪共享问题、CAS无锁操作）。

代码任务：在 Controller 中引入 Disruptor。当外部有大量请求涌入时，先放进 Disruptor 缓冲，然后消费者线程再通过之前写好的 Netty 引擎把任务下发给 Worker。完成这一步，你的调度系统就具备了“大厂级”的吞吐量。

🚀 第3周：异构计算，驱动 RTX 5090 的 Compute Worker
本周目标：让计算节点能够真正干活，利用 GPU 算力跑模型。

Day 1-2：认识 DJL (Deep Java Library)

学习任务：了解 Java 是如何通过 JNI/JNA 调用底层 C++ 甚至 CUDA 库的。阅读 DJL 官方文档，了解如何用纯 Java 跑 AI 模型。

代码任务：在本地搭建 DJL 环境（不需要马上用5090，普通电脑CPU也可以先跑通），写一个最简单的 Demo，比如加载一个预训练的图像分类模型或 ONNX 模型，并输入图片得到结果。

Day 3-4：Worker 任务状态机

代码任务：在 Worker 节点中维护任务状态（接收中 -> 等待显存 -> 推理中 -> 完成 -> 结果回传）。接收到 Controller 的任务指令后，将任务放入本地队列。

Day 5-7：亚毫秒级推理与显存管理

学习任务：了解模型预热（Warmup）和批处理（Batching）。

代码任务：优化 Worker 的逻辑。不能来一个任务加载一次模型，要在启动时把模型加载到内存/显存中复用。结合 Java 21 虚拟线程，实现异步等待 GPU 推理结果，完成后通过 Netty 传回 Controller。

💾 第4周：海量数据底座，自研分布式存储与系统整合
本周目标：解决大文件（如模型参数或海量推理数据）的存储问题，并把整个系统联调跑通。

Day 1-3：手写简易 HDFS (分布式文件存储)

学习任务：理解文件分片（Chunking）和副本机制（Replication）。

代码任务：

写一个文件切片工具，把一个大文件切成多个固定大小（如 64MB）的 Block。

Controller 记录“哪个文件的哪个 Block 存放在哪个 Worker 节点上”（元数据管理）。

Worker 节点提供接收 Block 并写入本地硬盘的功能。

Day 4-5：全面联调与异常测试

代码任务：把前3周的代码合并起来！

启动 Controller。

启动 2 个 Worker 节点。

提交一个大文件让其分布式存储；提交 10000 个推理任务，观察 Disruptor 缓冲、Netty 传输、DJL 推理是否顺利运转。

测试任务：拔掉一个 Worker 的网线（模拟宕机），看看 Controller 能不能发现并把任务重新调度给另一个 Worker（容灾测试）。

Day 6-7：写在简历上的亮点（Benchmarking）

代码任务：写压测脚本。记录你的系统“每秒能处理多少个并发请求（QPS）”、“零拷贝和不用零拷贝的延迟差距是多少”。把这些数据用图表画出来，写进你的项目 Readme.md 里。
